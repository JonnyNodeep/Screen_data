# Техническое задание (MVP)

## 1. Наименование проекта

OCR таблиц со скриншотов с последующим сохранением в Excel.

## 2. Цель и область применения

Разработать Python-скрипт, который:

- принимает папку с изображениями таблиц;
- извлекает текст;
- преобразует текст в структурированные данные;
- сохраняет результат в Excel-файл.

## 3. Входные данные

### 3.1 Формат входных файлов

Поддерживаемые форматы изображений:

- `.png`
- `.jpg`
- `.jpeg`

### 3.2 Характеристики входных данных

- скриншоты таблиц;
- фиксированная структура полей:
  - ФИО
  - Дата рождения
  - Пол
  - Город
  - Телефон
  - Email

### 3.3 Источник данных

Локальная папка `/images`.

## 4. Выходные данные

### 4.1 Формат результата

Excel-файл: `result.xlsx`.

### 4.2 Структура таблицы

Колонки в фиксированном порядке:

ФИО | Дата рождения | Пол | Город | Телефон | Email

### 4.3 Требования к выходным данным

- одна строка соответствует одному человеку;
- пустые значения: `null` или пустая ячейка;
- корректная кодировка: UTF-8.

## 5. Архитектура решения (MVP)

Последовательность модулей:

1. `Image Loader`
2. `OCR Module (PaddleOCR)`
3. `Text Preprocessing`
4. `GPT Parsing (OpenAI API)`
5. `Aggregation`
6. `Export (Excel)`

## 6. Функциональные требования по модулям

### 6.1 Image Loader

**Назначение:** загрузка изображений из директории.

**Требования:**

- читать все файлы из `/images`;
- выполнять фильтрацию по расширению;
- обрабатывать ошибки чтения (битые файлы).

**Выход:** список путей к изображениям.

### 6.2 OCR Module

**Назначение:** извлечение текста с изображения.

**Инструмент:** `PaddleOCR` (без `table mode`).

**Требования:**

- извлекать весь текст;
- сохранять порядок строк;
- объединять текст в одну строку.

**Выход:** строка текста (`raw OCR`).

### 6.3 Text Preprocessing

**Назначение:** подготовка текста для GPT.

**Операции:**

- удаление лишних переносов;
- нормализация пробелов;
- удаление мусорных символов.

**Выход:** очищенный текст.

### 6.4 GPT Parsing Module

**Назначение:** преобразование текста в структуру.

**Модель:** `gpt-4.1-mini`.

**Вход:** OCR-текст.

**Выход:** JSON-массив объектов.

**Формат объекта:**

```json
{
  "ФИО": "string",
  "Дата рождения": "string",
  "Пол": "string",
  "Город": "string",
  "Телефон": "string | null",
  "Email": "string | null"
}
```

### 6.5 Aggregation Module

**Назначение:** объединение данных.

**Требования:**

- объединять результаты всех изображений;
- хранить результат в списке.

### 6.6 Export Module

**Назначение:** сохранение в Excel.

**Инструменты:**

- `pandas`
- `openpyxl`

**Требования:**

- фиксированный порядок колонок;
- создание одного файла `result.xlsx`.

## 7. Основной алгоритм работы

1. Получить список изображений.
2. Для каждого изображения:
   1. выполнить OCR;
   2. выполнить очистку текста;
   3. выполнить GPT-парсинг;
   4. добавить результат в общий список.
3. Создать `DataFrame`.
4. Сохранить Excel-файл.

## 8. Обработка ошибок

### 8.1 OCR

Если текст пустой, файл пропускается.

### 8.2 GPT

Если получен невалидный JSON:

- повторить запрос 1 раз;
- при повторной ошибке пропустить.

### 8.3 Файловые ошибки

Ошибки должны логироваться.

## 9. Ограничения MVP

- отсутствует API;
- отсутствует параллельная обработка;
- отсутствует сложная валидация;
- отсутствует UI.

## 10. Критерии приемки (Definition of Done)

Система считается рабочей при выполнении всех условий:

- обрабатывает не менее 10 изображений подряд;
- формирует Excel без падений;
- не менее 80% строк распознаны корректно;
- структура таблицы соблюдена.

## 11. Расширения (не входит в MVP)

- table OCR (PP-Structure)
- FastAPI
- batch processing
- очередь задач
- валидация данных
- UI интерфейс

## 12. Нефункциональные требования

- простой CLI-запуск: `python main.py`;
- время обработки: примерно 2-5 секунд на изображение.
